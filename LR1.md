## Ход выполнения работы

### 1. Регистрация домена (опционально)

> В рамках данной работы домен `.ru` не регистрировался в связи с требованиями идентификации личности (необходимость предоставления паспортных данных). Вместо этого использовался прямой IP-адрес сервера.

### 2. Аренда VDS-сервера

Был арендован VDS-сервер у провайдера **Host-0** (u3498227) со следующими характеристиками:

- Операционная система: Ubuntu 22.04 LTS
- ОЗУ: 512 МБ
- SSD: 20 ГБ
- CPU: 1 ядро

**IP-адрес сервера:** `80.249.144.226`

### 3. Генерация SSH-ключей

На локальной машине с ОС Windows была выполнена генерация пары RSA-ключей командой:

```bash
ssh-keygen -t rsa -b 4096
```

Публичный ключ (содержимое файла id_rsa.pub) был скопирован на сервер через панель управления VDS в разделе «SSH-ключи пользователя root».

### 4. Подключение к серверу по SSH

Подключение выполнено из терминала PowerShell:

```bash
ssh root@80.249.144.226
```

### 5. Первичная настройка сервера

После входа были выполнены команды обновления пакетов:

```bash
apt update
apt upgrade -y
```

Система успешно обновлена, ошибок не возникло.

### 6. Установка веб-сервера NGINX

Установка произведена командой:

```bash
apt install nginx -y
```

Проверка статуса сервера:

```bash
systemctl status nginx
```

Результат: active (running).

### 7. Создание и редактирование HTML-страницы

Стандартная страница NGINX была удалена, а вместо неё создан файл /var/www/html/index.html с помощью редактора nano:

```bash
rm /var/www/html/index.nginx-debian.html
nano /var/www/html/index.html
```

Содержимое файла (использован английский язык для корректного отображения кодировки в терминале):

```bash
<!DOCTYPE html>
<html>
<head>
    <title>My Lab Work</title>
</head>
<body>
    <h1>Hello, Student!</h1>
    <p>My server is running at IP address 80.249.144.226</p>
    <p>This is my first web page on VDS.</p>
</body>
</html>
```
После сохранения файла выполнен перезапуск NGINX:

```bash
systemctl reload nginx
```
